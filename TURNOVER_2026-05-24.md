## Two regressions to fix first (in order)

### REG-1: `Score At Update` ALLEXCEPT bug — Page 3 score timeline scattered

Symptom (user-confirmed via screenshot, Service render): on Page 3 (RiskDetail) drillthrough, the "Score Over Time" line chart shows many overlapping markers spread horizontally across the entire date range instead of the selected risk's ~3-10 update dates. Same on both sides (current risk + next risk).

Root cause: `pbip/Tonnelle_Risk.SemanticModel/definition/tables/_Measures.tmdl` defines `Score At Update` (and `Score At Update Next`, `Score Change Display`, `Score Change Display Next`) with lookback DAX that uses `ALLEXCEPT(Risk_Updates, Risk_Register[risk_id])`. `Risk_Register[risk_id]` is not a column on `Risk_Updates`. ALLEXCEPT degrades to `ALL(Risk_Updates)` and removes the per-risk drillthrough filter. The lookback then sees all 128 updates across all 37 risks.

Fix: rewrite each of the 4 measures to use explicit risk_id scoping:
```dax
VAR ThisDate = MAX(Risk_Updates[update_date])
VAR ThisRisk = SELECTEDVALUE(Risk_Register[risk_id])
VAR PLookback =
    CALCULATE(
        MAXX(
            TOPN(1,
                FILTER(ALL(Risk_Updates),
                    Risk_Updates[risk_id] = ThisRisk
                    && NOT ISBLANK(Risk_Updates[probability_at_update])
                    && Risk_Updates[update_date] <= ScoreDate
                ),
                Risk_Updates[update_date], DESC,
                Risk_Updates[update_id], DESC
            ),
            Risk_Updates[probability_at_update]
        )
    )
... C, S lookbacks same shape
```

For the `_Next` measures, the equivalent is `ThisRisk = SELECTEDVALUE(Risk_Updates_AsNext[drillthrough_risk_id])` because Risk_Updates_AsNext has its own risk_id column.

Also fix the `PriorCount` block inside `Score Change Display` and `Score Change Display Next` — same ALLEXCEPT bug there for the "is this the first update" logic.
